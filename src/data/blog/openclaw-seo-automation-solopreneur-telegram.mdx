## The Stack: OpenClaw, n8n, and Telegram in One Place

Think of it as three layers:

1. **OpenClaw** – AI reasoning, research, and content generation. It has skills for Gmail, Slack, Google Sheets, and browser automation; you add custom tools or webhooks as needed.
2. **n8n** – Workflow engine. Schedules runs, moves data between tools, and holds API keys so OpenClaw never sees them. 400+ nodes (Sheets, Airtable, Telegram, etc.).

## Workflow 1: From Topic to Content Brief (Automated)

**Goal:** You send a topic or seed keyword (e.g. via Telegram or a Google Sheet). OpenClaw runs keyword research, clusters ideas, and produces a content brief with target keywords and outline.

**Setup:**

- OpenClaw with web search (e.g. Tavily or SerpAPI) and access to a sheet or database.
- n8n workflow: trigger = “new row in Sheet” or “Telegram command.” Payload goes to OpenClaw via webhook.
- OpenClaw (with the right skills) runs the research and writes the brief into the sheet or back to n8n.
- n8n sends a Telegram message: “Brief ready for [topic] – link to sheet.”

**Important:** Keep API keys (SerpAPI, OpenAI, etc.) in n8n environment variables or secrets; pass only the topic/keyword to OpenClaw. That way you get deep automation without exposing keys to the agent.

**Solopreneur angle:** Run this once per day for one new topic. In a month you have 30 briefs; pick the best 10–15 and write or generate first drafts. That is a content pipeline without a full-time writer.

## Workflow 2: Brief to Published Post (With Human Checkpoints)

**Goal:** From brief to published post with minimal hand-holding: research → outline → first draft → meta description → optional human edit → publish to CMS.

**Steps:**

1. n8n runs on a schedule (e.g. daily) or when a brief is marked “ready” in the sheet.
2. OpenClaw receives the brief and generates an outline; you can approve or skip (e.g. via Telegram “approve” button or sheet checkbox).
3. OpenClaw writes the first draft (length and tone set in the brief). Again, optional human checkpoint.
4. OpenClaw generates meta title, description, and suggested keywords.
5. n8n pushes the final content to your CMS (WordPress, Ghost, or a headless API) or drops it in a “review” folder.

**AdSense angle:** Structure briefs for E-E-A-T: “experience” mentions, clear sourcing, and readability. OpenClaw can add “According to …” and “Data from …” style lines; you add real experience in the edit pass. Telegram alert: “Draft ready for [URL] – approve to publish.”

## Workflow 3: Backlink Outreach You Approve from Telegram

**Goal:** You get a list of prospect URLs (from a sheet or a tool). OpenClaw drafts outreach; you send or semi-automate sending via n8n + Gmail/SMTP. Replies and status updates go to Telegram.

**Setup:**

- Sheet: columns for URL, contact email, status (pending / sent / replied), and notes.
- n8n: “New row” or “Daily” trigger → fetch prospect row → send to OpenClaw with context (your site, topic, offer).
- OpenClaw returns a short, personalised outreach line (or full email). n8n writes it to the sheet and optionally sends via Gmail node.
- When you (or a tool) mark “replied,” n8n can notify Telegram: “Backlink reply from [domain] – check sheet.”

**Why Telegram:** You can approve or reject drafts from your phone (“use draft 1” / “skip”), so you keep control without sitting in the sheet. Solopreneurs often use one Telegram group for “content” and one for “outreach” to avoid noise.

## Workflow 4: Your Daily SEO Digest (Delivered to Telegram)

**Goal:** Every morning you get a Telegram message: top 5 ranking changes, new AI Overviews for your keywords, and one competitor move.

**Setup:**

- OpenClaw (with ranking and search tools) runs on a schedule via n8n (e.g. 6:00 AM your time).
- It aggregates data from your tracking sheet or API (e.g. rank tracker, SEMrush, or manual SERP checks).
- OpenClaw formats a short digest (text or markdown). n8n sends it to your Telegram with a “View full report” link if you host reports elsewhere.

This is “SEO automation” in the sense of *automated reporting and awareness*; the actual ranking work is still your content and backlinks, but you stay informed without opening five tools.

## Keep It Safe: Keys in n8n, Human Checkpoints

- **No key in OpenClaw:** All paid APIs (SerpAPI, OpenAI, etc.) live in n8n. OpenClaw gets only the payload (e.g. keyword, URL) and returns results. That reduces risk of key leakage and keeps billing in one place.
- **Human checkpoints:** For content and outreach, use “approve before publish” or “approve before send” in n8n (e.g. wait for a Telegram reply or sheet update). That keeps quality and avoids spam.
- **Rate limits:** In n8n, add delays and retries so you do not hit API limits or get flagged for aggressive automation.

## When the Bot Won’t Run Your Commands

If OpenClaw does not run your Telegram command:

- Check that the Telegram channel is enabled in OpenClaw config and the bot token is correct.
- Use **webhook mode** for the bot on a VPS (not polling) so messages are delivered reliably.
- Ensure the command text is clear (e.g. “Research keyword: [keyword]” or “Send digest”) and that the agent has the right skills and tools enabled.

If n8n does not trigger: check webhook URL, firewall, and that the OpenClaw endpoint is reachable from n8n (same host or correct network).
